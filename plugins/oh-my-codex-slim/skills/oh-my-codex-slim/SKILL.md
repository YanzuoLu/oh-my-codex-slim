# Oh My Codex Slim

Use this skill when a task benefits from lightweight orchestration. Keep all Codex system, developer, approval, sandbox, and mode instructions authoritative.

Recommended flow:

1. Clarify the goal and constraints.
2. Use read-only investigation roles for discovery, docs, risk review, or verification when helpful.
3. Use `fixer` only for bounded implementation work that is allowed by the active mode.
4. Continue directly if subagents or tools are unavailable.

In Plan/read-only contexts, produce plans and read-only analysis only; do not delegate write-capable implementation.
